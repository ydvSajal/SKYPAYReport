2. Introduction and Problem Statement

2.1 Background

Cryptocurrency and blockchain technology have introduced new paradigms for financial transactions and value transfer. However, the complexity of managing multi-currency payments, exchange rate conversions, and settlement across multiple networks remains a significant barrier for Web3 teams.

Traditional payment solutions in the Web3 space are either fragmented across multiple tools or lack intelligent automation capabilities. Teams must manually coordinate payment requests, track transactions, reconcile accounts, and manage refunds—processes that are time-consuming, error-prone, and inefficient.

2.2 Problem Statement

Problem: Web3 teams lack a unified, intelligent payment infrastructure that can autonomously handle complex payment operations while maintaining integration with existing team workflows.

2.3 Solution Overview

AgenPay solves this problem by deploying AI agents that understand natural language payment requests and execute complex, multi-step payment workflows autonomously. The solution integrates three key components:

  • Intelligent Agent Layer: LangGraph-powered agents that process natural language, extract payment details, and coordinate multi-step operations
  • Payment Infrastructure: X402Pay protocol implementation for secure, non-custodial payment requests with cryptographic verification
  • Workspace Integration: Native Notion integration enabling teams to manage payments within their existing collaboration tools

2.4 Project Objectives

  1. Develop an autonomous AI agent capable of processing natural language payment requests
  2. Implement secure X402Pay protocol integration for non-custodial payment links
  3. Create seamless Notion API integration for workspace-first payment management
  4. Support multi-token payments with automatic exchange rate handling
  5. Provide comprehensive API documentation and RESTful endpoints
  6. Ensure enterprise-grade security with authentication, authorization, and encryption
  7. Deploy on Sei Network with support for mainnet and testnet operations

3.2 Component Interaction Flow

The typical flow for a payment request execution:

  1. User submits natural language payment request through dashboard or API
  2. Express.js API server authenticates request and validates JWT token
  3. Request routed to AgentPayAgent for processing
  4. Agent uses LLM to extract payment details and determine action path
  5. Agent executes appropriate tools: payment link creation, email notification, Notion synchronization
  6. X402PayService generates secure payment link and blockchain metadata
  7. NotionService synchronizes payment records to user workspace
  8. Response returned to client with payment tracking information
  9. Facilitator service monitors blockchain for payment confirmation
  10. Upon confirmation, system triggers webhook and updates payment status

5. Core Components and Services

5.1 AgentPayAgent (Payment Processing Agent)

The AgentPayAgent is the central intelligence component of the system, implementing a sophisticated LangGraph state machine with 1600+ lines of carefully structured code.

Key Functions:

  • extractPaymentDetails(): Uses regex patterns and heuristics to identify payment amounts, recipient addresses, currencies, and descriptions from natural language input
  • createPaymentLink(): Generates X402Pay payment links with cryptographic signatures and embedded payment metadata
  • createMultiTokenPaymentLink(): Enables users to pay with any token while merchants receive preferred currency through DEX aggregation
  • getPaymentOptions(): Queries supported tokens and current exchange rates from on-chain liquidity pools
  • sendPaymentEmail(): Composes and sends HTML email notifications with payment links, QR codes, and tracking information
  • saveToNotion(): Synchronizes payment records to Notion databases with automatic field mapping

LLM Fallback Chain:

The agent implements intelligent fallback between three AI models to ensure operational resilience:

  1. Primary: OpenAI GPT-4 (optimal for complex reasoning)
  2. Secondary: Google Gemini (extended context, ultra-large prompts)
  3. Tertiary: MiniMax (stable inference, backup operation)

5.2 NotionService (Workspace Integration)

The NotionService handles OAuth-based Notion integration, enabling seamless workspace synchronization.

Key Capabilities:

  • OAuth 2.0 token exchange for workspace authorization
  • Automatic Notion workspace structure creation (database setup, dashboard layout)
  • Real-time record insertion and updates
  • Bidirectional sync between payment system and Notion
  • Property mapping for custom database schemas

5.3 X402PayService (Payment Protocol Handler)

The X402PayService manages the X402Pay protocol implementation, providing secure payment request generation and verification.

Key Algorithms:

Payment Route Verification: Validates payment routes against active registered recipients, ensuring only authorized wallets can receive payments.

Native SEI Confirmation: Implements on-chain verification by:

  1. Fetching transaction details from Sei blockchain
  2. Verifying recipient and sender addresses match payment request
  3. Confirming payment amount matches expected value
  4. Marking payment as confirmed once all checks pass

5.4 WalletService (Multi-Signature Wallets)

Manages cryptocurrency wallets through Privy and Coinbase CDP integration, supporting:

  • Multi-user wallet access with role-based controls
  • Transaction signing and gas fee estimation
  • Balance tracking across multiple tokens and chains
  • Secure key management and custody

5.5 SchedulingService (Automation Engine)

Handles recurring payments, scheduled refunds, and automatic reconciliation tasks using cron expressions and task queues.

Supported Operations:

  • Daily/weekly/monthly recurring payment schedules
  • Automatic refunds for expired payment requests
  • Periodic reconciliation reports
  • Scheduled database cleanup and archival

5.6 OKXDexService (Token Swap Integration)

Provides DEX aggregation for multi-token payment support, enabling users to pay with any token while merchants receive their preferred currency.

8. Security Features

8.1 Authentication and Authorization

JWT-based Authentication: All API endpoints enforce JWT token validation. Tokens include user ID, expiration time, and cryptographic signature.

Password Security: User passwords are hashed using bcryptjs with 10 salt rounds before storage. Raw passwords are never stored or logged.

Role-Based Access Control: Users can only access their own payment records and wallet operations. Administrative operations require elevated permissions.

8.2 Transport Security

  • HTTPS/TLS: All production endpoints use TLS 1.3 encryption
  • CORS Protection: Cross-origin requests validated against whitelist
  • CSRF Prevention: SameSite cookie attributes and token validation
  • Helmet.js: Automatic HTTP security headers (X-Frame-Options, CSP, etc.)

8.3 Rate Limiting

express-rate-limit middleware enforces strict rate limiting:

  • 100 requests per 15 minutes per IP address
  • 10 failed login attempts lock account for 30 minutes
  • Payment creation limited to 50 requests per hour per user

8.4 Data Protection

  • Encryption at Rest: Sensitive database fields (API keys, private keys) encrypted using AES-256
  • Encryption in Transit: All network communication uses TLS 1.3
  • Input Validation: Zod schema validation on all request inputs
  • SQL Injection Prevention: Parameterized queries through Prisma ORM
  • XSS Prevention: HTML escaping in email templates and responses

8.5 Blockchain Security

  • Non-custodial Wallets: Privy handles key management, system never accesses private keys
  • Signature Verification: All blockchain transactions cryptographically signed by wallet owner
  • X402Pay Protocol: Payments require both request signature and transaction confirmation
  • Replay Attack Prevention: Transaction nonces and chain IDs included in signatures

8.6 Audit and Compliance

  • All API calls logged with timestamp, user ID, endpoint, and status code
  • Failed authentication attempts flagged and monitored
  • Database access logs maintained for compliance audits
  • Payment transactions immutable and traceable to blockchain

Security Checklist: See SECURITY_CHECKLIST.md for comprehensive security action items and compliance guidelines.
